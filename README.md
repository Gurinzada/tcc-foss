# 🇧🇷 Versão em Português (PT-BR)

## 🎓 TCC FOSS

Este projeto foi desenvolvido como Trabalho de Conclusão de Curso (TCC)
pela Universidade Federal de Lavras (UFLA), sob orientação do professor
Paulo Afonso Parreira Júnior.

Trata-se de uma aplicação web que analisa repositórios open source do
GitHub e gera insights tanto para novos contribuidores quanto para
mantenedores, avaliando se o projeto é amigável para iniciantes.

O foco é identificar boas oportunidades de contribuição por meio de
métricas objetivas, como issues para iniciantes, qualidade da
documentação e atividade da comunidade.

------------------------------------------------------------------------

## 🎯 Objetivo

Ajudar desenvolvedores iniciantes a responder perguntas como:

-   Este repositório é amigável para iniciantes?
-   Existem issues do tipo good first issue?
-   O projeto possui documentação básica?
-   A comunidade é ativa?
-   Qual é o tempo de resposta nas issues?

Tudo isso sem necessidade de autenticação, utilizando apenas dados
públicos da API do GitHub.

------------------------------------------------------------------------

## 🛠️ Tecnologias Utilizadas

### Frontend

-   React\
-   Vite\
-   Ionic\
-   Mantine UI\
-   Axios\
-   Redux Toolkit\
-   Bootstrap

### API

-   GitHub REST API v3

------------------------------------------------------------------------

## 🧩 Arquitetura

A aplicação funciona 100% no frontend, consumindo diretamente a API
pública do GitHub.

------------------------------------------------------------------------

## 🚀 Como Executar o Projeto

``` bash
cd frontend
npm install
npm run dev
```

Acesse: http://localhost:5173

# 🇺🇸 English Version (EN)

## 🎓 FOSS TCC

This project was developed as a Final Course Project (TCC) at the
Federal University of Lavras (UFLA), under the supervision of Professor
Paulo Afonso Parreira Júnior.

It is a web application that analyzes open source GitHub repositories
and generates insights for both new contributors and repository
maintainers, evaluating whether a project is beginner-friendly.

The main goal is to identify good contribution opportunities using
objective metrics such as beginner issues, documentation quality, and
community activity.

------------------------------------------------------------------------

## 🎯 Objective

Help beginner developers answer questions like:

-   Is this repository beginner-friendly?
-   Are there good first issues available?
-   Does the project have basic documentation?
-   Is the community active?
-   What is the issue response time?

All of this is done without user authentication, using only public data
from the GitHub API.
